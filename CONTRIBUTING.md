# Contributing to NewsDigest

Thank you for your interest in helping improve NewsDigest! We value all contributions—from bug reports to new features.

## 🚀 Getting Started

1.  **Fork** the repository on GitHub.
2.  **Clone** your fork locally:
    ```bash
    git clone https://github.com/YOUR_USERNAME/NewsDigest.git
    ```
3.  **Create a branch** for your feature or fix:
    ```bash
    git checkout -b feature/amazing-new-feature
    ```

## 🛠 Project Standards

### Code Style
- Follow the official [Swift Style Guide](https://www.swift.org/documentation/api-design-guidelines/).
- Use meaningful variable names and avoid abbreviations.
- Ensure all new features include appropriate documentation.

### Architecture
- Maintain the **MVVM** pattern used throughout the app.
- Keep UI components reusable and isolated in `Features/Shared`.
- Business logic should reside in the `Services` layer.

## 📮 Pull Request Process

1.  Ensure your code builds without warnings in Xcode.
2.  Update the `README.md` if your changes introduce new configuration requirements.
3.  Submit a Pull Request targeting the `main` branch.
4.  Provide a clear description of the changes and any relevant issue numbers.

## 🐞 Reporting Issues
- Use GitHub Issues to report bugs.
- Include steps to reproduce the issue and details about your environment (iOS version, device, etc.).
